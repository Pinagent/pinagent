# @pinagent/web

The Pinagent marketing site.

```bash
pnpm --filter @pinagent/web dev      # http://localhost:3030
pnpm --filter @pinagent/web build
pnpm --filter @pinagent/web start
```

Stack: Next.js 16 App Router, React 19, Tailwind 4. Shared design tokens come
from `@pinagent/ui` (the same theme the dock and example apps use).

This app is intentionally minimal — one static page (`app/page.tsx`) and one
client component for the install tabs (`app/_components/InstallTabs.tsx`).
`/docs` is a Server Component that reads the repo-root `README.md` at build
time. Favicons and OG cards are generated by `next/og` so brand colors stay
sourced from `@pinagent/ui/tokens`.

## Deploying to Vercel

This is a static Next.js app inside a pnpm + turbo monorepo. Vercel handles
the shape out of the box once the project is pointed at the right directory.

1. `pnpm dlx vercel link` from `apps/web/` — pick or create a Vercel project.
2. In the Vercel dashboard, set **Root Directory** to `apps/web`. Vercel
   auto-detects pnpm workspaces and runs `pnpm install` from the monorepo
   root before `next build`, so workspace deps like `@pinagent/ui` resolve
   correctly. No `vercel.json` needed.
3. Framework preset, build command, and output directory all auto-detect.
4. The `outputFileTracingRoot` set in `next.config.ts` anchors file tracing
   to the monorepo root so future serverless routes bundle workspace deps
   correctly. The `turbopack.root` setting on the same line silences the
   "multiple lockfiles detected" warning when the repo is checked out under
   a worktree.
